# Security Policy

## Supported Versions

Security fixes will generally be applied to the latest released version.

| Version        | Supported |
| -------------- | --------- |
| Latest Release | ✅         |
| Older Releases | ❌         |

## Reporting a Vulnerability

Please do **not** report security vulnerabilities through public GitHub issues.

Instead:

1. Open a GitHub Security Advisory (preferred), or
2. Contact the maintainer privately.

Please include:

* Description of the issue
* Steps to reproduce
* Potential impact
* Affected version(s)
* Suggested mitigation (if known)

## Response Process

We aim to:

* Acknowledge reports within 7 days
* Investigate and validate findings
* Develop and release fixes when necessary
* Credit reporters when appropriate

## Scope

Examples of security-related issues include:

* Arbitrary file access
* Command injection
* Path traversal vulnerabilities
* Unsafe handling of user input
* Accidental exposure of secrets

Thank you for helping keep the project secure.
